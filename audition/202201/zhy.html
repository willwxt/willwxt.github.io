    # corejava 40

    1. 合法的变量名: 允许26个字母(或其他国家文字),数字,下划线, dollar符, 数字不要开头
      • 其中, ￥属于中文
    2. 保存类的源文件名: 一个类中只能最多一个public类, 文件名需要与其保持一致, 而至于非public的类则没有约束
    3. java基本数据类型表述(表示)范围: byte - short - int - long - float - double
    4. 对于循环, 一种用法是使用label标记循环, 使用continue跳出循环至标记位置 (一般都不用好吧..)
    5. 对于三目运算符, 会和普通表达式计算一样, 会向大的表述范围进行转换
    6. 方法中的局部变量, 必须初始化后才能使用
    7. sout('5' + 2) 的结果是 55 ('5'对应的ASCII码是53)
    8. byte a = 128 纯数值字面量会根据表述范围表示为byte, short, int...类型, 如此从大到小的赋值, 必须进行强转; 而从小到大则不用 float f = 100L
    9. 对于switch, 要注意传入的变量类型有限 byte short int char String enum
    10. num = num++ 对于num, 其值不变
    11. 题目中的if等语句括号要小心, 很可能是没有的
    12. 对于java方法, 都是值传递; 引用也是引用(地址)值传递
    13. if(flag = true) 的写法是正确的
    14. 如果方法没有返回值必须声明为void (构造方法不用声明)
    15. 重载 是在一个类里面, 方法名字相同, 而参数不同. 返回类型可以相同也可以不同
    16. 重写是子类对父类的允许访问的方法的实现过程进行重新编写, 返回值和形参都不能改变
    17. 父类构造器不能被继承, 不能被重写; 构造器的访问控制修饰符和正常方法一样多种定义
    18. 避免菱形继承: java中类的单一继承, 接口则通过编译器检查是否有多个同名default方法
    19. super()必须在构造器的第一行调用; this()本类构造器也必须在第一行; 且两者是互斥的
    20. 抽象类不能创建对象
    21. 小心题目中的class同名普通方法, 并不是构造器!
    22. java内容加载顺序 先加载静态变量, 代码块(父类 > 子类); 再创建对象: 赋予默认值之后, 普通变量赋值, 代码块, 构造器(父类 > 子类)
    23. 对于类方法, 类属性, 不存在空指针异常; (针对使用null对象调用)
    24. 拥有对应的子类才能进行强转向下转型 (可能报错ClassCastException), 不然编译报错
    25. 多态: 有继承, 向上转型, 方法重写
    26. 属性没有多态的概念, 用的该对象的属性 (向上转型也不会用子类的属性)
    27. 对于方法重载参数父子类的情况下, 传入参数均匹配的情况下, 子类的方法优先调用; 对于均匹配但参数类型无继承关系的情况下, 调用处报编译错误(无法匹配);
    28. 所有异常的父类: Throwable
    29. catch子句的排列: 先子类后父类
    30. finally中修改内容会生效, 但是无法修改之前的返回值; 再加上一句return则可以覆盖前面的返回值;
    31. new String() 在堆中创建String对象, 并且指向方法区; String是特殊引用类型, 在方法传递中只传递了常量的地址;
    32. split方法参数中切片字符串具有正则含义, 属于正则匹配符的符号, 需要使用 \\ 进行转译
    33. substring方法返回新字符串, 原来的字符串是不变的
    34. StringBuffer没有重写equals, 其append方法会返回对象本身(this)
    35. finally在System.exit() 之后是不会执行的
    36. Math.round(-11.5) 结果是 -11 (加0.5并向下取整)
    37. 基本数据类型可以获得Class类对象: int.class
    38. 类的成员变量包括实例变量, 静态变量 (方法同理)
    39. 二维数组的定义中, 一维长度必须定义, 二维长度可后续定义
    40. java可以处理TCP和UDP
    41. 匿名内部类不能有构造器
    42. String s="welcome"+"to"+360; 字面量拼接会在编译期优化, 实际只生成一个常量
    43. 子类方法的访问权限不能低于父类
    44. 父类的构造器不能被子类继承
    45. split的目标字符串没有匹配, 就把整个字符串放至数组中第一个元素
    46. join()底层调用了wait(), 所以也释放锁资源
    47. threadlocalmap使用开放定址法解决冲突, hashmap使用链地址法解决
    48. 软引用内存满了之后会被清理; 弱引用,会被GC; 虚引用: 跟踪对象回收用, 正常情况没有任何用处
    49. 被动引用 (直接调用父类的static方法或字段), 多态下的子类是不会被加载的
    50. 泛型: JVM类型擦除; 但是可以通过反射获取到实际类型;
    51. >> 算术右移, >>> 逻辑右移
    52. 父类构造器多态调用时也会去找子类的方法, 但是这时子类还未初始化的话, 成员变量可能为null

    # 网络编程socket标准写法

    //服务端 listen和accept
public static void main(String[] args) throws IOException {
    ServerSocket serverSocket = new ServerSocket(PORT);
    while (true) {
    Socket socket = null;
    InputStream inputStream = null;
    OutputStream outputStream = null;
    try {
        //无数据时会阻塞在这里，有数据即accept事件会触发accept方法的调用
        //当一个连接进来，第二个连接进不来，因为单线程时，还阻塞在下面的read
        socket = serverSocket.accept();
        inputStream = new BufferedInputStream(socket.getInputStream());
        outputStream = new BufferedOutputStream(socket.getOutputStream());
        //读取正文内容
        byte[] flush = new byte[1024];
        int length = 0;
        StringBuffer rec = new StringBuffer();
        while ((length = inputStream.read(flush)) != -1) {
            rec.append(new String(flush, 0, length));
        }
        String back = "返回数据";
        //返回数据
        outputStream.write(back.getBytes(), 0, back.getBytes().length);
        outputStream.flush();
    } catch (Exception e) {
    	log.e(e);
    } finally {
    	close(socket,inputStream,outputStream);
    }
}

public static void doo() throws IOException {
    Socket socket = null;
    InputStream in = null;
    OutputStream out = null;
    String msg = "你好你好你好好好！";

        //发送数据
        socket = new Socket("127.0.0.1", 7777);
        out = new BufferedOutputStream(socket.getOutputStream());
        in = new BufferedInputStream(socket.getInputStream());
        out.write(msg.getBytes());
        out.flush();
        //任何的输入流或输出流的close()都会造成Socket关闭 但是不关闭又导致服务端无法接收到-1

        byte[] flush = new byte[1024];
        int length = 0;
        StringBuffer rec = new StringBuffer();

        socket.close();
    } catch (IOException e) {
        e.printStackTrace();
    } finally {
        close(socket, in, out);
    }
}

    # 范式

    表示数据库设计标准的级别 https://www.zhihu.com/question/24696366

    高级别的范式都对低级别的范式兼容
1NF: 数据库表的每个字段(属性含义)都不可再分
2NF: 不存在非主属性对于码的部分函数依赖
3NF: 消除了非主属性对于码的传递函数依赖
实际中,为了性能或者扩展的需要,经常做到2NF或者1NF; 但作为数据库设计人员,至少应该知道3NF的要求是怎样的

    # 事务传递注解

    Required:必须的。说明必须要有事务, 没有就新建事务。
supports:支持。说明仅仅是支持事务, 没有事务就非事务方式执行。
mandatory:强制的。说明一定要有事务, 没有事务就抛出异常。
required_new:必须新建事务。如果当前存在事务就挂起。
not_supported:不支持事务, 如果存在事务就挂起。
never:绝不有事务,否则抛出异常
nested:嵌套, 当前存在事务则嵌套, 否则就创建新事务来执行此方法

    # 加密算法：对称非对称

    对称秘钥加密: 加密和解密的秘钥是相同的;
非对称加密/公开密钥加密: 持有公钥的用户都可解密查看,但只有私钥持有者可以生成密文
DES: 对称算法,数据加密标准,速度较快,适用于加密大量数据的场合
3DES: 是基于DES的对称算法,对一块数据用三个不同的密钥进行三次加密,强度更高
RC2和RC4: 对称算法,用变长密钥对大量数据进行加密,比 DES 快
AES: 高级加密标准,对称算法,是下一代的加密算法标准,速度快,安全级别高
RSA：由 RSA 公司发明,是一个支持变长密钥的公共密钥算法,需要加密的文件块的长度也是可变的,非对称算法
DSA: 数字签名算法,是一种标准的 DSS（数字签名标准）,严格来说不算加密算法
MD5：严格来说不算加密算法,只能说是摘要算法

    git 分支
git branch $branch #新建分支, 但不切换当前分支;
git checkout -b $branch #新建分支并切换到该分支上;
git branch -d $branch #删除分支
git checkout $branch #分支切换(把HEAD指针指向目标分支)(HEAD是一个特殊指针, 永远指向当前分支)
git log --oneline --decorate --graph --all #查看分支情况
git merge $branch #将目标分支合并至当前分支; 当出现合并冲突Conflict, 使用 git add 标记冲突已解决, 最后使用commit提交本次合并
git rebase $branch #把当前分支rebase到目标分支上(找到两个分支的公共祖先,把当前分支所做的修改应用在目标分支上)
#合并和变基在结果上没有任何区别, 但是变基的分支历史记录是一条直线, 没有分叉
#变基还能实现对某一段提交进行复制应用
#变基的风险: 如果提交存在于你的仓库之外, 而别人可能基于这些提交进行开发, 那么不要执行变基

    # TCP UDP

    TCP是面向连接的, TCP建立连接:
1. A向B请求连接, 发送SYN(数据段标识); A变为SYN_SENT; B从LISTEN变为SYN_RECEIVED
2. B回复A, 发送ACK(确认收到)+SYN(数据段标识); B状态还是已收到SYN
3. A再回复B, 发送ACK(确认收到); A和B变为ESTABLISHED
断开连接
1. A向B请求断开, 发送FIN(完成发送的标志位); A变为FIN_WAIT_1; B变为CLOSE_WAIT
2. B同意, 发送ACK(确认收到); B变为LAST_ACK
3. B向A请求断开, 发送FIN; A变为TIME_WAIT
4. A同意, 发送ACK: A和B都变为CLOSED
