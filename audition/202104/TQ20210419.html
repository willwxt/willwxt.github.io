# Java中有哪些容器

首先根据线程安全分为普通容器和并发容器, 当然还有java数组对象(这里略过)

我们可以根据java接口分为 collection(包括list ,set ,queue) 和map (按顺序开始聊, 首先是list, set直接和map一起)

# 普通list:

线性表(可以顺序读写), 通用实现为ArrayList和LinkedList

# ArrayList

使用数组实现, 数组(连续结构, 下标索引快, 频繁修改导致数组全拷贝) 各种操作可以视为常量级

背书: 初始容量10, 扩容过程(先将旧容量扩大50%, 与目标容量比较, 判断是否越界, 最后数组拷贝)

# LinkedList

使用双向链表结构实现(头尾操作快, 索引慢, 添加删除快, 节点占用更多) 实现了队列/双向队列接口

# 特殊的list

考虑线程安全的:

  • Vector 由synchronized保证线程安全
  • CopyOnWriteArrayList 写时复制, 由lock保证线程安全, 由两个数组保证效率
  • Collections.SynchronizedList 装饰者模式, 对主要方法进行包装, 非包装的方法不保证线程安全

只读的 Arrays.asList(Integer[]..) 以这种方式生成的list不允许增删

# 普通的queue, deque都同理

  • 普通的queue其实就LinkedList和ArrayDeque(数组实现), 只不过操作不同(offer, poll, peek), 一般都不支持null值

# 特殊的queue

  • 优先队列 PriorityQueue 堆排序实现
  • Collections的AsLIFOQueue (栈) (push, pop)
  • 线程安全的 ConcurrentLinkedQueue

# blockingqueue

设计用于生产消费者队列, 线程安全, 对一些操作(插入, 删除, 检查), 拥有多种处理方式(抛出异常, 返回特殊值, 阻塞, 超时), 提供(take, put)

  • ArrayBlockingQueue: lock1+condition2(非空, 非满)实现, 可以指定int类型的队列长度, 同一把锁, 两个阻塞状态
  • LinkedBlokingQueue: lock2, condition2, take put两把锁(必要时还是两个一起锁的), 两个阻塞状态, 同样是int最大值的长度
  • DelayQueue 指定任务执行顺序, 按时间调度
  • SynchronousQueue 同步 生产者和消费者 的阻塞队列, 都需要阻塞等对方到...才能完成一次消费!
  • TransferQueue 增加了可以让生产者一直阻塞到消息消费完成的功能
  • 其他: 往下拓展到lock, aqs实现, 向上是线程池, deque就跳过了

# Map

保存键值对, key不可重复; 它提供了三个集合视图(key, value, entry); 一般不使用可变对象作为key, 因为对象属性变化可能导致equals不同
