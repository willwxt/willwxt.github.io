        # 概述

        高性能, 内存占用小, 并发能力强 50000 并发 (tomcat 200)

        # 功能

        • 反向代理 (正向代理)
        • 负载均衡
          • 轮询
          • 加权轮询. 权重 (按权重分发请求, 这样小机器也可上线)
          • ip hash (根据ip绑定session, 但是down机丢失session, 所以使用redis共享session而不建议使用Nginx实现)
        • 资源的动静分离

        # 安装

        # Windows

        直接下载, 解压, cmd 窗口运行 Nginx.exe 命令

        # Linux

        # 确保本机没有 nginx
whereis nginx
# tar 解压, ./configure 生成makefile make编译 make install安装到usr/local/nginx
yum -y install gcc-c++ zlib zlib-devel openssl openssl-devel pcre pcre-devel

        # 配置文件

        主要是 nginx.conf

        重新读取配置

        • 全局配置
        • event
          • 连接数, 事件, 性能相关配置
        • http
          • http全局配置
          • upstream配置(负载均衡配置, 供location中使用)
          • server(监听端口, 代理配置)
            • listen
            • location
              • proxy_pass 代理

        其他配置请百度

        # 命令

        nginx

        nginx -s stop

        nginx -s quit

        nginx -s reload

        # 踩坑

        • 代理名不能使用下划线 会报400
        • 安装完nginx后使用 -t 查看配置文件目录 不要改错了配置文件
        • 配置完成后响应速度变慢 不要使用localhost 最好使用ip 具体原因可以百度

        # docker部署nginx

        参考 https://www.pianshen.com/article/3910349119/

        # 下载镜像文件	# 如要安装别的版本,请自带版本号
docker pull nginx
# 随便跑下实例
docker run -p 80:80 --name nginx  -d nginx
# 将容器内的配置文件,html页面拷贝到当前目录 	# 找不到文件会报 403 forbidden
docker container cp nginx:/etc/nginx /mydata/nginx
docker container cp nginx:/usr/share/nginx/html /mydata/nginx
# 修改文件名称
cd /mydata/nginx
mv nginx conf
# 终止容器 # 删除原容器
docker stop nginx
docker rm $ContainerId
# 正式运行
docker run --name nginx -p 80:80 \
-v /mydata/nginx/html:/usr/share/nginx/html \
-v /mydata/nginx/logs:/var/log/nginx \
-v /mydata/nginx/conf:/etc/nginx \
-d nginx
