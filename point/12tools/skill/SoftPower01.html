    # 一些提问的回答

    • 如何解释离职?

    寻求个人发展, 原来公司也可以待下去, 但是你们公司肯定更好

    • 如果工作经历中有半年断档期

    和朋友创业, 我负责技术, 发现自己不适合创业, 决定还是好好找份工作... 但是我肯定不能这么说

    • 简历的重要原则

    简历决定了面试机会, 而不是技术决定. 所以写简历不要关注自己会什么, 而是关注如何写能拿到面试机会, 最后等简历完成, 在尝试去达到简历的80%

    • 简历上面推荐贴照片

    比较有亲和力, 背景是景色的那种半身照, 稍微PS, 看到简历就想要见下这个人

    • 站在别人的角度思考

    沟通时候要站在对方角度思考: 如果面试官看到这份简历是否想面试你?

    • 推荐35岁以上学mba

    • 面试不通过怎么办? 加强心理建设, 不要巨婴, 下一家继续面.

    • 最好的投递简历时间: 工作日9点左右(8-10点), 这是大多数HR精力充沛的工作时间

    • 简历邮件的标题: 请做一个标题党

    • 面试穿着: 阳光干净整洁, 管理好自己的第一形象, 表情管理

    • 面试回答: 可能不精确, 但是不要有怀疑语气, 需要多练习

    • 面试时间: 提前5-10分钟到即可

    • 模拟面试

    面对镜子模拟面试

    • 面试请求: 当电话接到请求可以先接受, 如果不去, 可以再通知对方, 沟通过程中不要怼人, 怼人你就会死

    • 离职交接: 很少有公司要你交接一个月, 没有一个你根本无所谓, 你留着反而影响项目组氛围

    • 试用期: 接到offer先后, 可以先进入试用期, 试用期是双方适应过程, 可以直接离职

    • 专业: 非本专业不用写, 培训要说是学校中或工作中学习的

    • 求职职位: 写一个

    • 鸡汤: 周围80-90%的人是不学习的, 只要坚持学习就能超越他们

    • 简历中的技能: 精通不要不敢写, java都不敢说精通谁还要你, 一般的JDK, 设计模式, JVM, 多线程都可以写

    • 加分项

    个人博客(喜欢总结和分享), GIT源码(热爱代码)

    个人评价, brbr

    个人经历, 获奖ACM参与奖... 参加XX开发/PMP/敏捷培训

    工作经历, 逆序

    项目经历: 重点***

    • 项目经历
    1. 不管原来项目使用的什么旧技术, 全部用新技术替换
    2. 业务的梳理,
    3. 技术上JVM/SQL优化根据实测达到什么样的效果提高了多少%, 架构想高并发方向去说
    4. 职责负责上描述, 问题, 解决方案, 制造亮点,
    • 一些其他

    被裁员 - n+1 - 劳动局. 因为n只计算底薪

    • 我这边没什么问题了, 你有什么问我的
    1. 技术面: 首先做好了解对方公司的功课, 聊公司的业务线, 用到哪些技术栈, 人员规模
    2. HR面: 不能问加班, 补助, 薪资, 问些五险一金, 上班时间, 无关痛痒的问题
    3. 谈薪水: 简历建议不写, 要写直接写目标薪资, 谈及上家公司薪水, 12*18, 发票补助(偷税)
    4. 猫腻: 用年终奖套路你要求的总年薪, 要了解年终奖的不确定性
    5. 和HR聊天带上录音笔... 面试老是面不过的带上录音笔...
    • HR提问, 你的理想, 职业规划, 优缺点

    理想, 正常说, 技术道路上.. 职业规划.. 刻苦, 学习/沟通能力还行..

    # 项目提问 - 把项目背出来, 对着镜子练

    1. 介绍一下这个项目 - 客户 预算 几个版本 第几期 多长时间 大概业务 业务名词压死
    2. 技术架构
    3. 项目有什么地方可以改进
    4. 项目有多大规模/ 多少人参与/ 分别是哪些/ 测试/开发/前端
    5. 您在项目中是什么角色, 介绍一下您负责的部分
    6. 数据库表结构设计, 字段, 类设计, Redis的运用细节
    7. 数据量有多大, QPS, 日活
    8. 并发量有多大
    9. JVM的问题
    10. 项目的心得(项目的改进)
    11. 加班的情况, 项目轶事
    12. 项目中的难点

    # 大厂面试

    说一个技术的时候, 最好连带着说出应用场景

    一致性哈希, 布隆过滤器, Paxos, 分布式一致性 -- 应用场景 优缺点 举例/画图说明

    说一个算法, 把类比的算法说出来, 两阶段提交, 三阶段提交, 2pc, 3pc, rap, 论文作者怎么想的

    讲java各种属性的时候结合版本说, 集合, 并发工具类变化比较多, 1.8, 1.9发生什么变化, jdk13 String多行定义

    面试官愿意问比较擅长的东西

    大厂面试的严格流程

    • 操作系统
    • 网络
    • SQL
    • 算法
    • java基础
    • 面试官手里有一张表

    面试官问一个知识点, 可以说其他知识点

    自问自答的方式

    尽量把知识讲得有趣一些... 讲述方式尽可能和网上的文章和视频不一样, 防止和别人冲突
