# Linux的目录结构
  • 基本介绍: linux文件系统采用级层树状目录结构, 最上层是根目录 /
  • 在Linux世界里, 一切皆文件
  • 规划好各个目录存放内容, 避免乱放文件
具体目录 含义
/bin Binary, 存放最经常使用的命令
/sbin SuperUser, 存放系统管理员使用的系统管理程序
/home 存放普通用户的子目录, 每个用户有一个自己的目录, 以账号命名
/root 系统管理员/超级用户的主目录
/boot 存放linux启动时使用的核心文件, 包括连接文件, 镜像文件
/proc 虚拟的目录, 系统内存的映射, 访问这个目录来获取系统信息
/srv service, 存放一些服务启动之后需要提取的数据
/sys 安装了linux 2.6内核中新出现的一个文件系统
/tmp 临时文件
/dev 设备管理器, 把所有的硬件用文件的形式存储
/media 自动识别U盘, 光驱, 并挂载到这个目录下
/mnt 让用户临时挂载别的文件系统, 可以将外部的存储挂载
/opt 额外安装软件摆放的目录, 如安装oracle数据库
/usr/local 另一个额外安装软件的目录, 一般通过编译源码方式安装的程序
/var 存放不断扩充的文件, 经常修改的文件, 如各种日志
/selinux SELinux是一种安全子系统, 控制程序只能访问特定文件

# rwx 权限详解

  • 作用到文件

    • r可读, w表示可写/可修改, 但不代表可以删除文件, 删除一个文件的前提是对该文件所在目录有写权限
    • x代表可执行

  • 作用到目录

    • r可读, 可以读取, ls查看目录内容
    • w可写, 可以修改, 目录内创建+删除+重命名目录
    • x可以进入该目录

  • chmod 修改权限

  • 第一种 , u所有者 g所有组 o其他 a所有

    • chmod u=rwx,g=rx,o=x 文件/目录 chmod o+w 文件/目录 chmod a-x 文件/目录

  • 第二种, 通过数字变更权限

    • 规则: r=4 w=2 x=1 rwx=7
    • chmod 751 文件/目录

# 5. crond任务调度

定时调用我们的脚步或者代码 crontab [选项] 简单任务可以不用写脚步

  • -e 编辑crontab定时任务 -l 查询任务 -r 删除当前用户所有任务 service crond restart 重启任务调度
  • 案例: 每小时的每分钟执行 */1 * * * * ls -l /etc >> /tmp/to.txt
项目 含义 范围
第一个 一小时当中的第几分钟 0-59
第二个 一天当中的第几个小时 0-23
第三个 一个月当中的第几天 1-31
第四个 一年当中的第几月 1-13
第五个 一周中的星期几 0-7(0和7星期天)
符号 含义
* 代表任何时间, 比如第一个*就表示一小时中每分钟都执行一次
, 代表不连续的时间, 比如 0 8,12,16 * * * 代表每天的8,12,16点
- 代表连续的时间范围,比如 0 5 * * 1-6 周一到六的凌晨5点
*/n 代表每隔多久执行一次, 比如 */10 * * * * 每隔10分钟

案例

时间 含义
45 22 * * * 每天 22:45
0 17 * * 1 每周一 17:00
0 5 1,15 * * 每月1号和15号的 05:00
40 4 * * 1-5 周一至周五的 04:40
*/10 4 * * * 每天的 04:00, 每隔10分钟
0 0 1,15 * 1 每1号和15号和每周一的 00:00, 最好日期和星期不要同时出现

  • linux中每个执行的程序都称为一个进程, 每个进程都分配一个ID号(PID)
  • 每个进程都会对应一个父进程(最大的init)
  • 进程有两种存在方式, 前台与后台
  • 一般系统的服务都是以后台进程的方式存在

# 终止进程kill和killall

如果某个进程执行一半, 或是消耗了很多系统资源, 需要停止, 使用kill

  • kill [选项] [PID] 通过进程ID杀死进程
  • killall [进程名称] 通过进程名称杀死, 支持通配符, 系统负载大时有用
  • 常用选项 -9 表示强制杀死进程
  • 案例
    • 踢掉非法登录用户 ps -aux | grep sshd 找到用户, 然后kill
    • 终止sshd /usr/sbin/sshd
    • 终止多个gedit编辑器 killall gedit
    • 强制杀掉一个终端 /bin/bash' 一个终端对应一个bash

pstree [选项] 树状显示进程 pstree -p pstree -u

# service 服务管理

端口--服务, 服务本质就是进程(守护进程), 通常监听某个端口, 等待其他程序请求

  • service 服务名 [start|stop|restart|reload|status]
  • centos 7 以后使用 systemctl
  • 防火墙 service iptables status
  • 查看有哪些服务
    • setup
    • ls -l /etc/init.d/
服务的运行级别
0 停机, 默认不能设置, 否则不能正常启动
1 单用户, root权限, 用于系统维护, 禁止远程登录
2 多用户状态, 没有NFS, 不支持网络
3 完全的多用户, 有NFS, 登录后进入控制台命令模式
4 保留
5 X11控制台, 登录后进入图形GUI模式
6 系统正常关闭并重启, 默认不能设置, 否则不能正常启动
  • 开机流程说明
    • 开机=>>BIOS=>>/boot=>>init进程=>>运行级别=>>运行对应级别需要自启动的服务
  • chkconfig 查看/设置服务各个运行级别
  • chkconfig --level 5 服务名 on/off 设置某服务在5级别下是否自启动
  • 重新设置后需要reboot生效
# 动态监控进程
  • top 显示正在执行的进程, 实时更新
  • -d 秒数 #指定每隔几秒更新, 默认是3秒 -i 不显示任何闲置/僵尸进程 -p [PID] 监控单个进程
  • 交互操作 P 以CPU使用率排序(默认) M 内存使用率排序 N PID排序
  • k 杀死进程 u 监控用户
  • 窗口显示
# 查看系统网络情况
  • netstat [选项] netstat -anp
  • -an 按一定顺序排列输出 -p 显示哪个进程在调用
  • 案例
    • netstat -anp | more 查看所有网络服务
    • netstat -anp | grep sshd
