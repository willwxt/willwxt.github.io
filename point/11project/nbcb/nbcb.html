      # 第一节课: 微服务概述, 简单的hello world

      # 第二节: Eureka, Gateway, Feign/RestTemplate 讲解

      • 概述:

        1. RESTFUL: HTTP返回码规定 400为客户端错误, 500为服务端错误
        2. 主键查询的, 推荐将主键写在url中; 基于属性的写在url的?后作为参数
        3. 简单描述了 zuul, ribbon, monitor, hystrix, turbine

      • Eureka

        • 简单概述: 分为实例instance, 客户端client, 服务端server, 三个维度
        • 调优参数 instance:
          • 多久没收到心跳的超时时间, 默认90秒, 生产30秒
          • client多久给server发送一次心跳: 默认30秒, 生产10秒
          • 是否使用ip地址: 生产true, 配合hostname使用
        • client:
          • 是否注册到注册中心: 本地开发可以false
          • 是否拉取服务列表: 肯定需要
          • 拉取服务列表的间隔: 默认30秒, 生产10秒或5秒
        • server:
          • 是否开启自我保护: 生产环境true
          • 清理无效服务的间隔: 默认60秒, 生产30秒

      • Gateway

        • 应用: 灰度发布, 服务迁移(新接口上线, 原接口继续提供服务, 给调用方足够时间进行迁移)

        • 流程: 请求, 匹配, 路由, 过滤, 转发到目标, 目标按原路径返回

        • 使用: 推荐修改网关日志级别, info无法看到转发日志

          reactor.ipc.netty：debug
org.springframework.cloud.gateway：trace
          1
          2

        • 断言: 是否转发, 控制用; 过滤器: 修改请求头, url

        • 应用:

          • 用filter实现计算请求耗时
          • 网关和注册中心组合使用, ribbon决定一个实例调用

      # 第三节: Ribbon, Hystrix, SpringAdmin, Apollo

      • Ribbon

        • 7种负载均衡策略: 轮询, 最小并发, 权重, 重试...

        • 使用apache abtest工具进行测试

          ab -n 1000 -c 1 http://localhost...	#用1个线程, 发送1000次
          1

        • 参数调优

          • 全局配置 && 单个微服务配置

          • 默认是开启重试, 可能会有重试报文, 行内接口幂等性没有保障, 目前只开启GET请求的重试

          • 连接超时, 读取超时时间, 生产配置

            ConnetTimeout: 2000
ReadTimeout: 5000 (包括重试的时间)
            1
            2

      • Hystrix

        • 隔离: 限制调用资源, 隔离不可用服务, 防止雪崩
        • 降级: 降级接口返回兜底数据
        • 熔断: 达到失败率后进行降级, 重试成功后恢复
        • 缓存: 合并请求机制 (生产没有用,说是对代码有侵入性)
        • 降级是客户端的策略, 和服务端无关
        • 熔断器状态: Closed(正常调用) - Open(调用降级) - Half-Open(一段时间会进入的半开状态)
        • 配置
          • Hystrix请求的超时时间, 一般小于Ribbon的读取超时
          • 在一个滑动窗口(默认10秒)中打开断路器的最少请求数
          • 将开启的断路器置为半开状态的等待时间
          • 错误百分比
          • 配置Hystrix请求线程池
          • 设置统计滑动窗口时间间隔
          • 设置滑动窗口被分成多少个桶bucket
        • 调优
          • 隔离方案: 核心数量,最大数量..
          • 线程隔离(会发生ThreadLocal数据丢失)
          • 信号量隔离: 最大信号量
          • Hystrix超时时间 5秒

      • Spring Boot Admin (行内用普罗米修斯)

        • 搭建, 查看服务状态线程数内存, 测量请求, 查看运行时环境, 查看真正加载的配置, 定时任务, 动态修改日志级别, JVM, 线程, HeapDump, Web, Audit

      • Apollo

        • 概述: 依赖, @EnableApolloConfig, 配置
        • 功能: 配置, 回滚, 灰度, 公共配置, 配置历史版本查看

      # 第四节: 水星平台

      # 第五节: 星云平台

      架构: 鄞州江北双中心双活(镜像副本)

      • 旧架构
        • 每个中心有四个域 网银APP, OA前置...
        • 每个域都有单独一套微服务架构: 注册中心, 网关, 属于该域的微服务
        • 域内调用，直接通过域内注册中心获取实例调用即可
        • 跨域调用，（路由网关将微服务代理注册到注册中心）通过路由网关进行跨域调用
        • 跨中心调用，根据全行注册中心转发调用
      • 新架构
        • 目的是去除的注册中心，只有全行注册中心，实现直接通过注册中心调用

      坑: 在星云空间站注册过程中域选择错误，此时脚本已经派发（Apollo配置），需要重新手动修改

      边车 sidecar：一个系统需要一个边车进行代理（路由转发），服务被代理注册到注册中心 (目前用的少，还在试点，后续主推)

      接口限流是使用guava令牌桶算法实现的

      水星星云平台新老系统traceid兼容问题(skywalking和sleuth) (OpenTracing规范)

      页面调用，注解跳过token校验，安全性请自己控制
