      # 最差时间复杂度

      额外空间复杂度

      • 时间复杂度相等时判断算法的优劣 --拼常数项 (放弃理论分析, 生成随机数据测试)

        • 插入排序好于冒泡排序

      • 常见时间复杂度排序

        • O(1) O(logN) O(N) O(NlogN) O(N^2) O(N^K) O(2^N) O(K^N) O(N!)

      # 最优解:

      保证时间复杂度尽可能低同时空间复杂度低 但: 不包括常数时间优化

      # 二分法

      • 必要条件: 只要能正确构建左右两侧的淘汰逻辑

      • 思路: 砍掉一半 注意: 边界处理

      • 题目

        • 有序列表查询

          • L + ((R - L) >> 2) 等价于(L+R)/2 且避免溢出

        • 满足>=n的最小位置

        • 局部最小值

      # 位运算

      • 异或运算(无进位相加)

        • 性质 0 ^ N = N N ^ N = 0 满足交换律和结合律

        • 不用额外变量交换两个数 (通过交换律解释)

        • 只一个数出现了奇数次, 其他都是偶数次

          • 答案为所有数异或的结果

        • 提取int类型数的最右侧的1 (二进制)

          • a & (-a)

        • 两个数出现了奇数次, 找到这两个数

          • 最终异或结果为 a ^ b 结果肯定不为0, 那么取最右侧出现的位置 通过这个位置为 0 或 1 , 将原数组分为两类数据 结果: 通过该位置状态来异或到两个数据中

        • 有一种数出现K次, 其他数都出现了M次 M > 1, K < M

          • 创建int[32], 每位存储所有数对应二进制位数的出现次数

      # 链表

      • 翻转单链表/双向链表

      • 删除指定元素(可能换头)

        • head先来到第一个不需要被删除的位置, 再后续

      # 栈和队列

      • 用双向链表实现

      • 用数组实现

        • 栈 - 数组+位置

        • 队列 - 循环数组

          • begin指向开始, end指向结尾 size控制是否可以增加

      • 实现特殊的栈, 额外返回栈中最小值

        • 数据栈 + 最小栈; 同步记录

      • 用栈实现队列

        • push栈 + pop栈 push栈必须一次性倒完 pop栈必须拿完才能倒数据 封装倒数据函数 f()

      • 用队列实现栈

        • 两个队列来回倒, 每次留最后一位返回 来回倒通过互换引用指向轻松实现

      • 图中用栈实现宽度优先, 队列实现深度优先

      # 归并排序

      • 递归

        • 取中点, 左边有序, 右边有序, 最后merge
        • merge : 简单的合并有序表

      • 迭代 : 通过2次方的步长进行循环调整

        • 注意: 边界处理, 步长溢出

      • 求小和(总和)问题

        • 一边merge, 一遍求小和出现的次数, 不断累加

      • 求所有逆序对问题

      • 大于两倍问题

      • 返回arr中, 累加和在[a, b]区间内的所有子数组

        • 前置知识: 前缀和数组
        • 首先转化为: 求以对应下标n结尾的符合条件子数组 其次转化为: 因为(前缀和数组), 故0-n前缀和已知, 记为sum 条件: (sum) - (x前缀和) ∈ [a, b] 区间, 等价于 (x前缀和) ∈ [sum-b, sum-a] 求 x
        • 使用merge sort, 求下标n右侧有多少位置达标
        • merge过程中一定是顺序的, 使用滑动窗口(双指针)

      # 快速排序

      • 荷兰旗问题

        • 给定x, 重构无序数组为:

      1. 小于等于x | 大于x

      2. 小于x | 等于x | 大于x

         - 一个区域: 小于等于区域
 - 两个区域: 小于区域, 大于区域
 - 根据情况扩张区域 / 移动下标

      • 快速排序1.0 (基于1)

      • 快速排序2.0 (基于2)

        • 以R划分, 递归调用partition(荷兰旗)

      • 快速排序3.0 随机快排

        • 与2.0区别在于 随机选择一个数放在R位置作为划分值
        • 最优情况是划分值在中间, 最差情况是在两侧 所以时间复杂度是概率期望 O(nlogn) 空间复杂度是概率期望 O(logn)
        • 迭代代码: 通过栈存放partition范围实现
